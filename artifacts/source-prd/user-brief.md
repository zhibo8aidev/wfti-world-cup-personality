# 用户原始需求整理 - WFTI 世界杯球迷人格大鉴赏

## 1. 需求背景与目标
- 背景：2026 世界杯赛期（6 至 7 月），直播吧需要一个高参与度 H5 互动活动承接赛事流量峰值，提升站内用户活跃度。
- 目标：
  - 活动期间参与测试 UV ≥ 500 万
  - 测试完成率 ≥ 75%
  - 结果页分享率 ≥ 15%
  - 通过站内联动拉动赛事内页日活 UV 提升 5%
- 范围：APP 内 WebView H5 活动，不涉及客户端原生改动。站内推送、竞猜彩蛋等联动由相关业务模块配合落地。

## 2. 页面与状态范围
1. 活动落地页
2. 答题页
3. 结果计算过渡页
4. 结果页
5. 分享海报

主链路：活动入口 -> 落地页 -> 16 题答题 -> 结果计算过渡页 -> 结果页 -> 分享海报

## 3. 页面需求摘要
### 3.1 活动落地页
- 标题：WFTI — 世界杯球迷人格大鉴赏
- Slogan：你不是在看世界杯，你是在演你自己。
- 副说明：16 道题，3 分钟，测出你的世界杯球迷人格密码
- 固定底部主 CTA：开始鉴赏
- 已完成用户额外显示：查看我的结果，主 CTA 改为重新测试
- 未登录用户点击开始鉴赏时，先拉起 APP 登录弹窗，登录成功自动进入答题页
- 首屏需骨架屏；加载超过 3 秒显示 loading 动效
- CTA 点击后置灰 0.5 秒防抖
- 不支持下拉刷新
- 网络异常展示错误兜底页
- 活动未开始 / 已结束分别展示状态页，其中已结束页需展示全站热门人格 TOP3

### 3.2 答题页
- 共 16 题，固定顺序展示，不随机
- 每题结构：题号、题干、A/B/C 三选一
- 顶部展示进度条与 N/16 计数
- 选项高亮 0.3 秒后自动切下一题
- 切题动效：当前题左滑出，新题右滑入，400ms
- 支持返回上一题并保留选择状态
- 第 16 题完成后自动进入结果计算过渡页
- 答题中断时，本地缓存 24 小时；重进需提示继续或重新开始
- 网络异常、缓存读取失败均需有兜底

### 3.3 结果计算过渡页
- 答题完成后展示 2 至 3 秒计算动效
- 期间前端完成算法计算，不依赖后端

### 3.4 结果页
- 页面结构：人格代号+中文昵称 -> 人物形象插画与标签云 -> 球迷人格解析 -> 球星对照/天敌/拍档 -> 二维码 -> 底部悬浮操作栏
- 底部操作栏：保存相册、分享名片、重新预测
- 结果持久化：将 UID + 人格代号 + 8 维评分 + 时间戳异步上报后端，同一用户重测覆盖旧结果
- 结果上报失败不影响展示，需本地缓存并按 2s/4s/8s 静默重试 3 次
- 计算异常默认回退到 MOOD，并记录异常日志

### 3.5 分享海报
- 前端使用 Canvas / html2canvas 生成 PNG 长图
- 海报包含：人格代号、中文昵称、一句话标签、八维雷达图、画像文案、球星对照、二维码、logo
- 二维码需要带 UTM：source=poster&personality={代号}
- 支持渠道：微信好友、朋友圈、微博、QQ、保存到相册
- 保存图片需申请相册权限，拒绝授权时提示去设置开启
- 海报生成中显示 loading，超过 5 秒失败提示并允许重试
- 连续 3 次生成失败后，隐藏生成海报按钮，仅保留保存截图提示
