# Product Spec - WFTI 世界杯球迷人格大鉴赏

## 1. 产品概述
直播吧世界杯活动 H5，运行于 APP 内 WebView。用户通过 16 道固定题目完成球迷人格测试，获得结果页与分享海报，并将结果异步上报后端用于留存与联动。

## 2. 业务目标
- 参与测试 UV ≥ 500 万
- 测试完成率 ≥ 75%
- 结果页分享率 ≥ 15%
- 拉动赛事内页日活 UV 提升 5%

## 3. 用户主流程
1. 用户从 APP 多入口进入活动落地页
2. 点击“开始鉴赏”
3. 未登录则先拉起登录弹窗，成功后自动进入答题页
4. 完成 16 题答题
5. 进入 2 至 3 秒结果计算过渡页
6. 查看结果页
7. 选择保存相册 / 分享名片 / 重新预测
8. 生成海报后调起系统分享面板或保存到相册

## 4. 页面需求
### 4.1 活动落地页
#### 展示内容
- 标题：WFTI — 世界杯球迷人格大鉴赏
- Slogan：你不是在看世界杯，你是在演你自己。
- 说明文案：16 道题，3 分钟，测出你的世界杯球迷人格密码
- 固定底部主按钮：开始鉴赏
- 若当前登录用户已有历史结果，增加“查看我的结果”，主按钮文案改为“重新测试”

#### 交互规则
- 页面加载先展示骨架屏
- 加载超过 3 秒时显示 loading 动效
- CTA 点击后按钮置灰 0.5 秒防重复点击
- 不支持下拉刷新
- 未登录点击开始鉴赏时需先拉起宿主登录弹窗

#### 异常处理
- 网络异常：展示“网络开了小差，点击重试”及重新加载按钮
- 活动未开始：展示“活动即将开始，敬请期待”
- 活动已结束：展示“活动已结束，感谢参与”及全站热门人格 TOP3

### 4.2 答题页
#### 基础规则
- 共 16 题，按固定顺序展示
- 每题包含：题号、题干、3 个选项（A/B/C）
- 每题分值 A=1，B=2，C=3，仅用于算法计算，不展示给用户

#### 交互规则
- 顶部展示进度条，包含百分比与 N/16
- 答完一题，进度条 300ms ease-out 动画增长
- 用户选择选项后，当前选项高亮 0.3 秒，再自动切入下一题
- 切题动画：当前题左滑出，新题右滑入，时长 400ms
- 支持点击左上角返回按钮回到上一题，保留已选状态
- 第 16 题完成后自动跳转结果计算过渡页

#### 中断与恢复
- 答题中关闭页面时，答案与当前进度缓存到 localStorage
- 24 小时内重进，提示“检测到未完成的测试，继续上次进度？”并提供继续 / 重新开始
- 超过 24 小时自动清理缓存

#### 异常处理
- 网络断开：Toast 提示“网络异常，请检查网络后重试”，已答数据保留
- 缓存读取失败：静默从第 1 题开始

### 4.3 结果计算过渡页
- 时长 2 至 3 秒，建议默认 2.5 秒
- 展示计算类动效与“正在分析你的球迷DNA……”文案
- 期间前端执行：8 维聚合、等级映射、隐藏人格优先检测、常规人格向量匹配

### 4.4 结果页
#### 页面结构
从上到下依次为：
1. 人格代号 + 中文昵称
2. 人物插画区域（含标签云）
3. 球迷人格解析
4. 球星对照 / 天敌 / 拍档
5. 二维码与品牌区
6. 底部固定悬浮操作栏

#### 展示规则
- 人格代号大字居中，昵称次级展示
- 插画采用极客蓝扁平化风格
- 标签云展示一句话标签组
- 球迷人格解析区域展示完整说明文案
- 页面支持纵向滚动
- 天敌与拍档不可点击跳转

#### 底部操作栏
- 第一行并列按钮：保存相册、分享名片
- 第二行居中小字链接：重新预测

#### 数据持久化
- 结果对象至少包含：人格代号、中文昵称、8 维原始分、等级向量、答案、时间戳、是否隐藏人格
- 结果页渲染不依赖后端返回
- 结果生成后立即异步上报后端，与用户 UID 绑定；同一用户重测覆盖旧结果
- 上报失败时本地缓存结果并按 2s / 4s / 8s 静默重试 3 次
- 若最终失败，放弃重试但不影响用户体验
- 若算法异常无法匹配人格，兜底返回 MOOD 并记录异常日志

### 4.5 分享海报
#### 生成规则
- 前端使用 Canvas / html2canvas 生成 PNG 长图
- 推荐宽度 750px，高度自适应，目标约 1334px
- 截取结果页内容区时自动隐藏底部操作栏
- 海报包含：人格代号、昵称、一句话标签、八维雷达图、画像文字、球星对照、二维码、logo

#### 分享与保存
- 生成完成后调起系统分享面板
- 目标渠道：微信好友、朋友圈、微博、QQ、保存到相册
- 二维码需带参数：source=poster&personality={代号}
- 保存图片时需申请相册权限
- 拒绝授权时 Toast 提示“请到设置中开启相册权限”
- 保存成功时 Toast 提示“图片已保存”

#### 异常处理
- 生成中显示 loading 蒙层与“海报生成中…”
- 超过 5 秒未完成，提示“生成失败，请重试”并关闭蒙层
- 连续失败 3 次后，隐藏生成海报按钮，仅保留保存截图提示

### 4.6 视觉语言与版式约束
#### 全局背景与头图衔接
- 页面需参照头图进行设计。
- 全局背景色必须严格吸附头图底部像素色，确保头图与正文背景无视觉断层。
- 页面主体应呈现连续影棚地板式延展感，避免出现上下分层、切片式拼接或背景突变。

#### 组件触感系统
- 交互组件采用“纳米级触感系统”方向。
- 主按钮必须包含微渐变、大半径弥散投影与内高光勾勒，体现精致拟真触感。
- 按钮至少给出默认、按下、禁用、加载中 4 种状态定义。
- 组件阴影、圆角、描边与高光强度需统一，不允许各模块各自为政。

#### 排版秩序
- 在明亮通透的全局调性下，正文、标题、说明文、按钮文案均需保持高对比度。
- 通过细粒度字间距调节建立工业级阅读秩序感，避免轻飘、松散或泛白。
- 重点数字、人格代号、按钮文案应具备更强层级与更稳定的视觉锚点。

#### 执行前提
- 若后续收到实际头图文件，以头图真实底部像素色为最高优先级。
- 目前该要求已作为设计约束写入文档，但实际头图素材仍待补充入库。

## 5. 数据与状态
### 5.1 本地缓存
- 答题草稿：答案、题号、最近保存时间、版本号
- 最近结果：人格结果对象，用于“查看我的结果”与上报失败兜底
- 草稿与结果分开存储，避免互相覆盖

### 5.2 服务端上报
- 字段建议：uid、personality_code、raw_scores、level_vector、answers、computed_at、is_hidden
- 上报时机：结果页展示后异步发送
- 同一 uid 新结果覆盖旧结果

## 6. 埋点建议
至少覆盖：
- 活动页曝光 / CTA 点击
- 登录拉起 / 成功 / 取消
- 每题曝光 / 选项点击 / 回退点击
- 恢复答题弹窗曝光 / 继续 / 重开
- 结果计算完成
- 结果页曝光
- 分享点击 / 成功 / 取消
- 保存相册点击 / 授权结果 / 成功 / 失败
- 重新预测点击
- 海报生成失败
- 结果上报失败 / 重试 / 放弃

## 7. 当前待补齐项
- 每个人格的球星对照、天敌、最佳拍档映射
- 标签云文案明细与插画素材
- 已结束态 TOP3 数据来源与展示口径
- 登录 / 分享 / 相册 / deep link 的 bridge 协议
- 后端结果上报接口文档与异常日志接收方式
- 实际头图文件或可访问链接，用于像素级吸色与视觉衔接校准

## 8. 验收基线
- 用户可从落地页完整走通 16 题 -> 过渡页 -> 结果页 -> 海报流程
- 24 小时内支持继续上次答题
- 算法结果严格遵循 question-algorithm spec
- 登录态、分享、保存相册、活动状态页与异常页均可正确触发
- 结果上报失败不影响展示，且静默重试符合口径
