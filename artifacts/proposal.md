# WFTI - 世界杯球迷人格大鉴赏 Proposal

## 项目背景
2026 世界杯赛期是直播吧赛事流量峰值窗口。项目目标是在 APP 内 WebView 落地一个高参与度 H5 互动活动，以“16 题测世界杯球迷人格”为核心玩法，承接流量高峰、提升站内活跃，并通过结果页分享与站内联动放大传播。

## 业务目标
- 活动期间参与测试 UV ≥ 500 万
- 测试完成率 ≥ 75%
- 结果页分享率 ≥ 15%
- 通过赛后推送、竞猜联动等站内协同，带动赛事内页日活 UV 提升 5%

## 产品范围
### 本期包含
- APP 内 WebView H5 活动落地页
- 16 题固定顺序答题页
- 结果计算过渡页
- 结果页
- 分享海报生成
- 本地断点续答、最近结果缓存
- 前端人格计算引擎
- 登录拉起、分享面板、保存相册、结果异步上报等接入点
- 活动未开始 / 已结束 / 网络异常等状态页

### 本期不包含
- 客户端原生改动
- 独立站外网页版本
- 人格详情二级页
- 复杂后台运营配置系统
- 推送、竞猜彩蛋等业务模块本身的改造工作（仅预留联动能力与埋点口径）

## 核心方案
### 1. 产品形态
- 载体：直播吧 APP 内 WebView H5
- 设备：移动端优先
- 结果计算：全部前端完成
- 结果存储：结果页先展示，再异步上报后端，不阻塞用户流程

### 2. 核心玩法
- 用户从多入口进入活动页，点击开始鉴赏
- 完成 16 道题后进入 2 至 3 秒结果计算过渡页
- 前端完成隐藏人格优先检测与常规人格向量匹配
- 输出人格结果页，并支持生成分享海报
- 用户可保存海报或走系统分享面板扩散

### 3. 数据与接入原则
- 题库、人格文案、隐藏人格规则、人格原型向量打包在前端静态配置中
- 结果计算逻辑完全前端化，不依赖服务端计算
- 用户 UID + 人格代号 + 8 维原始分 + 时间戳由前端异步上报后端
- 登录、分享、相册权限、深链跳转通过宿主 bridge / JSBridge 能力接入

## 依赖与协同边界
- 客户端 / 宿主：登录弹窗、分享面板、保存相册、深链 / 应用商店跳转能力
- 后端：结果上报接口、活动状态接口、异常日志接收、已结束页 TOP3 数据来源
- 运营 / 内容：活动正式命名、二维码链接、分享文案、视觉素材、人格插画、球星对照 / 天敌 / 拍档数据
- 其他业务模块：赛后推送、竞猜联动彩蛋等联动执行

## 关键产品决策
- 保持 H5 方案，不引入原生开发依赖
- 以固定 16 题 + 隐藏人格优先 + 向量匹配作为唯一算法基线
- 断点续答有效期为 24 小时
- 结果页分享与海报生成作为核心传播杠杆
- 结果上报失败不影响前台展示，采用本地缓存 + 静默重试兜底

## 当前风险
- 结果页中的球星对照、天敌、拍档、标签云明细仍未补齐
- 活动已结束态的全站热门人格 TOP3 数据来源待确认
- 登录 / 分享 / 保存相册 / 深链能力依赖宿主 bridge 协议
- Canvas / html2canvas 在低端机型上存在性能与兼容风险
- 指标目标较高，需要配合入口投放与站内联动节奏共同达成
